üéØ Routing Modes Explained

The system supports 5 routing modes for flexible migration:

1

legacy-only (Default)

All requests go to the current TDS API. Use this mode for stable operations.

PUT /api/config/routing-mode { "routingMode": "legacy-only" }
2

both (Dual Mode)

Executes on BOTH APIs in parallel. Compares responses for validation. Use this for testing and validation before migration.

PUT /api/config/routing-mode { "routingMode": "both" }
3

shadow

Legacy API is used for responses, but Salesforce API runs in background for testing (responses ignored).

PUT /api/config/routing-mode { "routingMode": "shadow" }
4

forwarding

Gradually shift traffic to Salesforce API based on percentage (e.g., 25% Salesforce, 75% legacy).

PUT /api/config/forwarding-percentage { "percentage": 25 }
5

salesforce-only (Complete Migration)

All requests go to Salesforce EWC API. Use this after successful validation.

PUT /api/config/routing-mode { "routingMode": "salesforce-only" }

Create JupixWebhook.js to handle system-specific webhook format

2

Create API Integration

Create JupixIntegration.js to fetch data from Jupix API

class JupixAPIClient { async fetchTenancyData(tenancyId) { // Jupix-specific API calls // Return standardized format } }
3

Add Organization Mapping

Configure Jupix agency ‚Üí TDS member mapping

{ "jupix-agency-456:DEFAULT": { "tdsMemberId": "TDS003", "tdsBranchId": "MAIN", "tdsApiKey": "jupix-api-key" } }
